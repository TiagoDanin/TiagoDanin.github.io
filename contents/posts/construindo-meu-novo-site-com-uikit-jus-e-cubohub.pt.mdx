### Um poucos sobre as três ferramentas.

Jus é uma incrível framework que permite em pouco tempo construir um site funcionar de modo semelhante ao Jekyll, só que com um sistema mais fácil de usar e eu gosto do jeito que ele trabalha com os modelos de paginas.

Já o UIKit para min é "o novo Bootstrap". Você tem os componentes com um padrão bem simples e limpo.

Crie o CuboHub para atualizar os sites dos meus projetos de forma automática.

### O começo!

Bom meu antigo website usava um tema que baixei na internet e fiz algumas modificações nele. Depois de um tempo a quantidade de projetos aumentaram e eu comecei a ter problemas de compatibilidade em alguns dispositivos, também tive dificuldades para manter as páginas sempre atualizadas.

Dei o primeiro passo, comecei a criar um gerador para isso, chamado de CuboHub disponível no Github: [https://github.com/CuboHub](https://github.com/CuboHub). A página é atualizada toda fez que tem uma nova modificação no repositório. Em seguida eu comecei a planeja meu novo website encontrei as ferramentas certas para o trabalho UIKit e Jus, com o layout em minha cabeça eu fui capaz de executar esse projeto em dois dias. O resultados você pode ver aqui tiagodanin.github.io.

PS: Jus usar o lobars, uma ferramenta incrível!

### Vantagens

#### Jus

*   Diversos formatados de arquivos suportados (é realmente incrível a quantidade).
*   Fácil de usar.
*   URLs limpas.
*   Suportado em GitHub Pages.
*   Muitas opções/funções para fazer modelo de página.

#### UIKit

*   CSS limpo.
*   Responsive web design.
*   Suporte para plataformas mobiles.
*   RTL suporte.
*   Usar prefixo "uk-" para tudo. :) Sem conflito com seu css.

#### CuboHub

*   Eu sou o desenvolvedor :v.
*   Acompanhar seus commits.
*   Fácil de configurar.
